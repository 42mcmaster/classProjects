# Teach-Back Peer Critique Guide

Peer Review & Rubric for HTML, CSS, and Intro JavaScript Teach-Backs

This guide is used when students teach their assigned content block and teammates evaluate them.
Teams of **three** will rotate: **Presenter → Reviewer → Reviewer**.

---

## Purpose

This activity helps you:

1. Practice teaching technical content clearly
2. Receive helpful peer feedback
3. Improve speaking, pacing, accuracy, and communication
4. Strengthen mastery of HTML/CSS/JavaScript concepts

---

## Deliverables

Each reviewer must:

* Complete the **Peer Review Rubric** (10 points total)
* Provide **three pieces of feedback** using the feedback frame
* Participate in group discussion after all three have presented

---

## Required Areas of Evaluation

| Area                            | What to Look For                                                    |
| ------------------------------- | ------------------------------------------------------------------- |
| **Speaking & Delivery**         | Clear voice, faces audience, doesn’t read slides, confident posture |
| **Content Accuracy & Coverage** | Required concepts taught, correct explanations, examples used       |
| **Clarity & Engagement**        | Easy-to-follow explanations, checks for understanding, visuals help |
| **Slide Quality**               | Organized, readable, minimal text, visuals/code support teaching    |
| **Professionalism & Timing**    | Within time limits, smooth pacing, followed the assignment          |

---

## Peer Evaluation Rubric (10 Points Total)

| Category                | Description                                       | Points |
| ----------------------- | ------------------------------------------------- | ------ |
| **Speaking & Delivery** | Clear voice, eye contact, didn’t read from slides | **/2** |
| **Content Accuracy**    | Required concepts taught correctly                | **/2** |
| **Clarity & Examples**  | Used examples and explained ideas clearly         | **/2** |
| **Slide Quality**       | Readable slides, visuals/code helped learning     | **/2** |
| **Teaching Presence**   | Timing, engagement, confidence                    | **/2** |

**Total Score:** `/10`

---

## Feedback Requirements

After scoring, each reviewer must give verbal and written feedback using this frame:

### ✅ One Thing You Did Well

(Example: *Your explanation of semantic tags was really clear.*)

### ✅ One Thing to Improve

(Example: *Try not to face the board as much — face the class.*)

### ✅ One Specific Suggestion

(Example: *Add one more code example to show how the `<form>` works.*)

---

## Team Rotation Procedure

1. **Presenter teaches** for ~5 minutes
2. **Reviewers score** using the rubric
3. **Reviewers give feedback** using the three-part guide (Feedback Requirements)
4. Rotate roles
5. Continue until all three have presented

---

## Quick Reviewer Checklist

* [ ] Speaker was clear and easy to understand
* [ ] Concepts were correct and complete
* [ ] Examples were included (code, visuals, analogies)
* [ ] Slides were readable and helpful
* [ ] Presenter engaged the audience
* [ ] Presenter stayed within the time limit
* [ ] Feedback provided using the required format
* [ ] Rubric completed
